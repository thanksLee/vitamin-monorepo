## 로그인 관련 표준단어 사전

아래는 로그인 관련 구현 가이드와 표준단어, 용어사전을 기반으로 작성한 논리 모델 입니다.

데이터 표준화에 맞게 로그인 유스케이스를 위한 논리적 데이터 모델을 테이블 형태로 작성했습니다. 테이블 접두사(prefix)는 각 테이블의 역할에 따라 다음과 같이 적용했습니다:

### 1. AUTH_USR (사용자 인증 테이블)

| 영문명               | 한글명             | 컬럼명 (영문) | 컬럼명 (한글)      | 데이터 타입 | 길이 | PK/FK/CK | Default  | 개인정보 | 암호화 | 설명                                             |
| :------------------- | :----------------- | :------------ | :----------------- | :---------- | :--- | :------- | :------- | :------- | :----- | :----------------------------------------------- |
| User Identifier      | 사용자 식별자      | USR_ID        | 사용자\_식별자     | VARCHAR     | 20   | PK       | -        | Y        | N      | 사용자를 고유하게 식별하는 값                    |
| User Password        | 사용자 비밀번호    | USR_PWD       | 사용자\_비밀번호   | VARCHAR     | 100  | -        | -        | Y        | Y      | 사용자 인증에 사용되는 암호화된 비밀번호         |
| User Status          | 사용자 상태        | USR_STAT      | 사용자\_상태       | VARCHAR     | 10   | -        | 'ACTIVE' | N        | N      | 사용자 계정의 현재 상태 (ACTIVE/INACTIVE/LOCKED) |
| Password Error Count | 비밀번호 오류 횟수 | PWD_ERR_CNT   | 비밀번호*오류*횟수 | INT         | -    | -        | 0        | N        | N      | 비밀번호 입력 실패 횟수                          |
| Login Date Time      | 로그인 일시        | LGN_DT_TM     | 로그인\_일시       | TIMESTAMP   | -    | -        | -        | N        | N      | 마지막 로그인 날짜와 시간                        |

### 2. AUTH_TKN (인증 토큰 테이블)

| 영문명          | 한글명        | 컬럼명 (영문) | 컬럼명 (한글)  | 데이터 타입 | 길이 | PK/FK/CK | Default | 개인정보 | 암호화 | 설명                           |
| :-------------- | :------------ | :------------ | :------------- | :---------- | :--- | :------- | :------ | :------- | :----- | :----------------------------- |
| User Identifier | 사용자 식별자 | USR_ID        | 사용자\_식별자 | VARCHAR     | 20   | FK       | -       | Y        | N      | 사용자를 고유하게 식별하는 값  |
| Refresh Token   | 갱신 토큰     | RFSH_TKN      | 갱신\_토큰     | VARCHAR     | 200  | -        | -       | N        | Y      | 접근 토큰 갱신에 사용되는 토큰 |
| Login Date Time | 로그인 일시   | LGN_DT_TM     | 로그인\_일시   | TIMESTAMP   | -    | -        | -       | N        | N      | 토큰 발급 날짜와 시간          |

### 3. SEC_BLK_LST (보안 차단 목록 테이블)

| 영문명              | 한글명           | 컬럼명 (영문) | 컬럼명 (한글)    | 데이터 타입 | 길이 | PK/FK/CK | Default | 개인정보 | 암호화 | 설명                       |
| :------------------ | :--------------- | :------------ | :--------------- | :---------- | :--- | :------- | :------ | :------- | :----- | :------------------------- |
| IP Address          | IP 주소          | IP_ADDR       | IP\_주소         | VARCHAR     | 15   | PK       | -       | Y        | N      | 차단된 IP 주소             |
| Login Attempt Count | 로그인 시도 횟수 | LGN_ATPT_CNT  | 로그인*시도*횟수 | INT         | -    | -        | 0       | N        | N      | 해당 IP의 로그인 시도 횟수 |
| Block Date Time     | 차단 일시        | BLK_DT_TM     | 차단\_일시       | TIMESTAMP   | -    | -        | -       | N        | N      | IP가 차단된 날짜와 시간    |

### 4. CFG_USR_SET (사용자 설정 테이블)

| 영문명           | 한글명        | 컬럼명 (영문) | 컬럼명 (한글)  | 데이터 타입 | 길이 | PK/FK/CK | Default | 개인정보 | 암호화 | 설명                                 |
| :--------------- | :------------ | :------------ | :------------- | :---------- | :--- | :------- | :------ | :------- | :----- | :----------------------------------- |
| User Identifier  | 사용자 식별자 | USR_ID        | 사용자\_식별자 | VARCHAR     | 20   | PK, FK   | -       | Y        | N      | 사용자를 고유하게 식별하는 값        |
| Theme Setting    | 테마 설정     | THM_SET       | 테마\_설정     | VARCHAR     | 10   | -        | 'LIGHT' | N        | N      | 사용자가 선택한 UI 테마 (LIGHT/DARK) |
| Language Setting | 언어 설정     | LANG_SET      | 언어\_설정     | VARCHAR     | 2    | -        | 'KR'    | N        | N      | 사용자가 선택한 UI 언어 (KR/EN/ZH)   |

### 5. LOG_LGN_ATPT (로그인 시도 로그 테이블)

| 영문명                   | 한글명             | 컬럼명 (영문) | 컬럼명 (한글)      | 데이터 타입 | 길이 | PK/FK/CK | Default | 개인정보 | 암호화 | 설명                               |
| :----------------------- | :----------------- | :------------ | :----------------- | :---------- | :--- | :------- | :------ | :------- | :----- | :--------------------------------- |
| Login Attempt Identifier | 로그인 시도 식별자 | LGN_ATPT_ID   | 로그인*시도*식별자 | INT         | -    | PK       | -       | N        | N      | 로그인 시도를 고유하게 식별하는 값 |
| User Identifier          | 사용자 식별자      | USR_ID        | 사용자\_식별자     | VARCHAR     | 20   | FK       | -       | Y        | N      | 사용자를 고유하게 식별하는 값      |
| IP Address               | IP 주소            | IP_ADDR       | IP\_주소           | VARCHAR     | 15   | -        | -       | Y        | N      | 로그인 시도한 IP 주소              |
| Login Date Time          | 로그인 일시        | LGN_DT_TM     | 로그인\_일시       | TIMESTAMP   | -    | -        | -       | N        | N      | 로그인 시도 날짜와 시간            |
| Login Attempt Status     | 로그인 시도 상태   | LGN_ATPT_STAT | 로그인*시도*상태   | VARCHAR     | 10   | -        | -       | N        | N      | 로그인 시도 결과 (SUCCESS/FAIL)    |

### 테이블 간의 관계:

1. AUTH_USR 테이블은 사용자의 기본 정보와 인증 관련 정보를 저장합니다.
2. AUTH_TKN 테이블은 AUTH_USR 테이블의 USR_ID를 외래 키로 참조하여 각 사용자의 갱신 토큰 정보를 저장합니다.
3. SEC_BLK_LST 테이블은 차단된 IP 주소 정보를 독립적으로 관리합니다.
4. CFG_USR_SET 테이블은 AUTH_USR 테이블의 USR_ID를 외래 키로 참조하여 각 사용자의 UI 설정 정보를 저장합니다.
5. LOG_LGN_ATPT 테이블은 AUTH_USR 테이블의 USR_ID를 외래 키로 참조하여 모든 로그인 시도 기록을 저장합니다.

이 데이터 모델은 사용자 인증, 보안, 설정, 그리고 로그인 시도 기록을 관리하는 데 필요한 기본적인 구조를 제공합니다. 개인정보 보호를 위해 필요한 컬럼에 대해 암호화를 지정했으며, 각 테이블과 컬럼에 대한 명확한 설명을 포함하고 있습니다.
